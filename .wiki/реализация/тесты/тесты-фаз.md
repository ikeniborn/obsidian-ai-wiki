# Тесты фазовых функций (tests/phases/)

Unit-тесты для всех фазовых функций: `runIngest`, `runInit`, `runLint`, `runQuery`. Тестируют поведение через сбор событий из AsyncGenerator.

## Основные характеристики

- **Расположение:** `tests/phases/`
- **Файлы:** `ingest.test.ts`, `init.test.ts`, `lint.test.ts`, `query.test.ts`

### Тесты runIngest (ingest.test.ts)

Проверяют:
- Ошибка при отсутствии аргументов
- Ошибка при пути вне vault
- Ошибка при отсутствии домена
- Успешная запись wiki-страниц из JSON-ответа LLM
- Блокировка записи страниц вне wiki-папки домена
- `source_path_added` event после успешного ingest
- `detectDomain()` — prefix-match по source_paths

### Тесты runInit (init.test.ts)

Проверяют:
- Ошибка при пустом domainId
- Ошибка если домен уже существует с entity_types
- `--dry-run` возвращает результат без `domain_created`
- Нормализация `wiki_folder` (убирает `vaults/<vaultName>/` prefix)
- `domain_created` event с vault-relative wiki_folder
- `ensureRootFiles`: создаёт `_schema.md`, `_index.md`, `_log.md` при их отсутствии
- `ensureRootFiles`: не перезаписывает существующие файлы

### Тесты runLint (lint.test.ts)

Проверяют:
- Ошибка при несуществующем домене
- Обработка нескольких доменов
- Результат содержит отчёт

### Тесты runQuery (query.test.ts)

Проверяют:
- Ошибка при пустом вопросе
- Ошибка при отсутствии домена
- Генерация `result` события с ответом

## Применение в контексте реализации

Тесты используют TDD-паттерн: mock LLM возвращает заготовленный JSON-ответ, тест проверяет события AsyncGenerator. Это позволяет тестировать логику фаз независимо от реального LLM и Obsidian API.
