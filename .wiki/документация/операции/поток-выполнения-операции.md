# Поток выполнения операции

Описание пути прохождения запроса от действия пользователя в UI до получения результата в боковой панели obsidian-llm-wiki.

## Основные характеристики

### Основной поток

```
User (нажимает кнопку)
  → LlmWikiView (ingestActive / query / lint / init)
  → WikiController
      │ [если isBusy()] → Notice("операция уже выполняется")
      │ [иначе]
      ├── создаёт AbortController
      ├── buildAgentRunner(vaultRoot)
      ├── setRunning(op, args) → View
      └── run(RunRequest) → AsyncGenerator<RunEvent>
            ↓ (for await ev)
          AgentRunner → phases/*
          phases → LlmClient.chat.completions.create(...)
          LlmClient → stream chunks
          phases → yield RunEvent
          RunEvent → WikiController → View.appendEvent(ev)

  Специальные события:
    domain_created    → settings.domains.push / saveSettings()
    source_path_added → consolidateSourcePaths() / saveSettings()

  По завершении: history.push(entry) / saveSettings() / View.finish(entry)
```

### Claude CLI Backend: парсинг потока

При использовании `claude-agent` бэкенда:

1. `ClaudeCliClient` запускает `iclaude.sh` через `spawn(..., {stdio: ["ignore","pipe","pipe"]})`
2. Читает stdout построчно через `readline`
3. Каждую строку передаёт в `parseStreamLine()` → `RunEvent | null`
4. При отмене: SIGTERM → grace 3000ms → SIGKILL

## Применение в контексте obsidian-llm-wiki

Весь поток выполнения операции обеспечивает live-обновление UI: пользователь видит шаги инструментов (Read, Write) в реальном времени по мере их завершения агентом.
