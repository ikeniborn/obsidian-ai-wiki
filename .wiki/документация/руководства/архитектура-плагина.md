# Архитектура плагина obsidian-llm-wiki

Obsidian-плагин для AI-powered компаундируемой базы знаний. Запускает AI-агента (Claude CLI или нативный OpenAI-совместимый LLM), отображает прогресс в боковой панели в реальном времени.

## Основные характеристики

### Операции плагина

| Операция | Описание | Таймаут по умолчанию |
|----------|----------|----------------------|
| ingest | Извлечение сущностей из файла, обновление wiki-страниц | 300 сек |
| query | Ответ на вопрос по базе знаний | 300 сек |
| query-save | Ответ на вопрос с сохранением результата в vault | 300 сек |
| lint | Проверка качества wiki-страниц домена | 900 сек |
| fix | Автоматическое исправление по результату lint | 900 сек |
| init | Начальная инициализация домена из source_paths | 3600 сек |
| chat | Интерактивный диалог с агентом (контекст lint/query) | 900 сек |

### Бэкенды

Плагин поддерживает два взаимозаменяемых бэкенда (Strategy pattern):

| Бэкенд | Описание | Клиент |
|--------|----------|--------|
| claude-agent | Запускает iclaude.sh как дочерний процесс, парсит stream-json stdout | [[claude-cli-client\|ClaudeCliClient]] |
| native-agent | Обращается к OpenAI-совместимому API (Ollama, OpenAI) | openai npm package |

### Ключевые архитектурные ограничения

- **Single-flight**: одновременно разрешена только одна операция
- **History limit**: 20 записей в settings.json (компромисс UX vs размер)
- **Wiki domains**: источник истины — `DomainEntry[]` в settings (персистируется Obsidian)
- **Source paths**: `consolidateSourcePaths()` убирает дубли и поглощённые потомки

## Применение в контексте obsidian-llm-wiki

### Слои архитектуры

| Слой | Модули |
|------|--------|
| Application (оркестрация) | main.ts, controller.ts |
| Infrastructure (I/O, LLM) | agent-runner.ts, claude-cli-client.ts, phases/ |
| Presentation (UI) | view.ts, settings.ts, modals.ts |
| Shared / Domain | types.ts, domain-map.ts, vault-tools.ts, stream.ts, source-paths.ts, i18n.ts |

### Ключевые архитектурные решения

1. **Single-Flight Guard** — одновременно разрешена только одна операция ([[wiki-controller|WikiController]].current). Причина: `iclaude.sh` не реентерабелен.

2. **AsyncGenerator Event Stream** — операции возвращают `AsyncGenerator<RunEvent>`, что позволяет передавать события в реальном времени в UI без колбэков и SharedState.

3. **Backend Strategy Pattern** — `LlmClient` — тонкий интерфейс (одно поле `chat.completions.create`). Две реализации: ClaudeCliClient и OpenAI.

4. **Domain-Driven Config** — все wiki-домены хранятся в `settings.domains: DomainEntry[]`, персистируются через Obsidian (`saveData`/`loadData`).

5. **Phases as Pure Generators** — каждая операция — отдельный файл в `src/phases/`, принимает параметры и возвращает `AsyncGenerator<RunEvent>`. Нет глобального состояния.
