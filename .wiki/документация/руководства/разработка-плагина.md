# Руководство разработчика

Руководство по сборке, установке и проверке работоспособности плагина obsidian-llm-wiki в процессе разработки.

## Основные характеристики

### Сборка

```bash
npm install
npm run build        # production build → main.js
npm run dev          # watch mode (esbuild)
npm test             # vitest (one-shot)
npm run test:watch   # vitest watch
```

Перед каждой сборкой (`npm run build`) необходимо автоматически поднимать patch-версию в `package.json` и `manifest.json`.

### Установка в vault (разработка)

```bash
ln -s $(pwd) ~/.config/obsidian/Plugins/obsidian-llm-wiki
```

Включить плагин: Settings → Community plugins → Installed plugins → LLM Wiki.

## Smoke-test чеклист

Набор проверок, выполняемых после каждой сборки:

| № | Проверка | Что проверяется |
|---|----------|----------------|
| 1 | Сборка и установка | `npm run build` без ошибок, symlink в vault, плагин включён |
| 2 | Settings | Заполнен `iclaudePath`, настройки сохраняются после перезагрузки |
| 3 | Open panel | Команда `LLM Wiki: Открыть панель` показывает side-panel |
| 4 | Ingest активного файла | Шаги Read/Write в панели, финальный отчёт виден |
| 5 | Query inline | Ответ появляется в панели |
| 6 | Query + save | После завершения Obsidian открыл созданную страницу |
| 7 | Cancel | Статус `cancelled` в истории при отмене в первые 2с |
| 8 | Lint | Выбор домена через модал, отчёт виден |
| 9 | Ошибки | Очистка пути к бэкенду → Notice |
| 10 | Race / single-flight | Параллельный вызов → Notice «Уже выполняется» |
