# Методология суррогатных ключей в витринных слоях

Набор принципов применения суррогатных ключей (surrogate keys) в витринных слоях (Data Mart) хранилища данных Ростелеком.

## Основные характеристики

| Принцип | Описание |
|---------|----------|
| Область применения | Суррогатный ключ нужен только при операциях Merge (upsert) |
| Внешние ключи | Не используются для обновления записей |
| Joins со справочниками | Допустимо через натуральные ключи, без суррогатного ключа |
| Направление развития | Рассматривается переход витринных слоёв на числовые ключи |

## Применение в контексте Ростелеком

Суррогатный ключ в витринных слоях оправдан только когда требуется операция Merge — в остальных случаях избыточен. Джойны со справочными таблицами реализуются через натуральные (бизнес) ключи, что снижает зависимость от генерации суррогатов.
