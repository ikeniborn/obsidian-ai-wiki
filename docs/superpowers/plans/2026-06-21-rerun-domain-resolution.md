# Re-run Domain Resolution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-running a query from the history list runs it against the query's original domain (its stored `domainId`), never silently falling back to the first domain.

**Architecture:** A new pure module `src/rerun-domain.ts` validates a history entry's stored `domainId` against the current domain list and returns an explicit success/failure (never a default). The history re-run handler in `src/view.ts` calls this helper and passes the resolved `domainId` straight to `controller.query`, instead of laundering it through the DOM `<select>` (which silently drops invalid values, causing the `domains[0]` bug). On failure it shows a `Notice` and runs nothing.

**Tech Stack:** TypeScript, Obsidian plugin API, esbuild (bundler + headless eval runner), ESLint.

## Global Constraints

- **Out of scope — do NOT change:** the `domains[0]` fallback in `src/phases/query.ts:51`; the command-palette query path in `src/main.ts` (intentionally runs with no `domainId`); the `detectDomain` (ingest) and `DomainModal` (`src/modals.ts`) `domains[0]` fallbacks.
- **i18n:** any new message key MUST be added to all three locales — `en` (default), `ru`, `es` — in `src/i18n.ts`, or `tsc` fails on the missing key in the typed locale objects.
- **Module placement:** the pure helper lives in its own file `src/rerun-domain.ts` (mirrors `src/source-deletion.ts`), NOT inside `src/view.ts`, so the headless eval imports a tiny type-only-dependency module instead of the whole plugin graph.
- **Testing model:** the resolvable logic is unit-tested headlessly via the esbuild out-of-vault eval pattern. The DOM/Obsidian re-run handler is verified by `tsc` + `lint` + manual steps (it cannot be unit-tested headlessly).
- **tsc baseline is NOT clean** (~135 pre-existing errors in untouched files). Gate on *new* errors in touched files only — never on "tsc clean".
- **Branch:** `dev/rerun-domain-resolution` (already created and checked out). Do not commit to `master`.

Create `src/rerun-domain.ts`:

```ts
import type { DomainEntry } from "./domain";
import type { RunHistoryEntry } from "./types";

export type RerunDomainResult =
  | { ok: true; domainId: string }
  | { ok: false; reason: "missing" | "not-found" };

/**
 * Resolve the domain a history entry should re-run against. The entry's stored
 * domainId is authoritative — this never falls back to the first domain. Returns
 * an explicit failure (not a default) when the id is absent or no longer exists,
 * so callers surface an error instead of querying the wrong domain.
 */
export function resolveRerunDomain(
  entry: RunHistoryEntry,
  domains: DomainEntry[],
): RerunDomainResult {
  const id = entry.domainId;
  if (!id) return { ok: false, reason: "missing" };
  if (!domains.some((d) => d.id === id)) return { ok: false, reason: "not-found" };
  return { ok: true, domainId: id };
}
```

Find this block (in `renderHistory`, currently lines 1199-1209):

        const rerunBtn = row.createEl("button", { text: "↺", attr: { title: "Re-run" } });
        rerunBtn.addEventListener("click", (e) => {
          e.stopPropagation();

Replace it with:
```ts
      if (it.operation === "query") {
        const rerunBtn = row.createEl("button", { text: "↺", attr: { title: "Re-run" } });
        rerunBtn.addEventListener("click", (e) => {
          e.stopPropagation();
          void (async () => {
            const domains = await this.plugin.controller.loadDomains();
            const r = resolveRerunDomain(it, domains);
            if (!r.ok) { new Notice(i18n().view.rerunDomainMissing); return; }
            if (this.domainSelect) {
              this.domainSelect.value = r.domainId;
              this.domainSelect.dispatchEvent(new Event("change"));
            }
            void this.plugin.controller.query(it.args[0] ?? "", r.domainId);
          })();
        });
      }
```
Note: the domain now reaches `controller.query` directly via `r.domainId`. The `domainSelect` is updated only for UX (to reflect which domain ran); it is no longer the source of truth, so a stale/empty select can no longer divert the query to `domains[0]`.
