# Community-review warning remediation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Clear two non-blocking Obsidian community-review warnings — the `undici` dependency advisory and the direct `fs`-module filesystem access — without removing any feature.

**Architecture:** Two independent surgical changes. (1) Raise the bundled `undici` version past the patched line. (2) Replace the single `fs.access` binary check with a `child_process` spawn probe (`<path> --version`), relocated next to the existing spawn code in `claude-cli-client.ts` and reused by the settings "Test connection" button. The probe is covered by an out-of-vault eval that spawns a real binary.

**Tech Stack:** TypeScript, esbuild (CJS bundle → `dist/main.js`), Obsidian plugin API, Node `child_process`, undici (bundled, desktop-only proxy), eslint-plugin-obsidianmd. Out-of-vault eval harness = esbuild bundle with `--alias:obsidian=<stub>`.

## Global Constraints

- `undici` version floor: **`^6.27.0`** (closes `GHSA-p88m-4jfj-68fv` + the 3 other undici advisories `npm audit` lists in the `<= 6.26.0` range).

- Node builtins must stay **lazy** (`await import("node:child_process")`) and **desktop-guarded** (`Platform.isDesktopApp`) — mirrors the existing `_generate` in `claude-cli-client.ts` and satisfies `npm run lint`.

- The custom-proxy feature (`src/proxy.ts`, ProxyAgent) and the Claude-CLI backend stay intact.
- `tsc` gate: no **new** errors in touched files. There are ~135 pre-existing errors in untouched files; do **not** gate on a clean `tsc` run, only on new errors in `src/settings.ts` and `src/claude-cli-client.ts`.

```typescript
/**
 * Out-of-vault eval for probeClaudeBinary (src/claude-cli-client.ts). Spawns a
 * REAL binary (the running Node executable) — no vault, no Obsidian, no LLM.
 *
 * Build & run (from repo root):
 *   node_modules/.bin/esbuild eval/claude-probe/run.ts \
 *     --bundle --platform=node --format=cjs \
 *     --alias:obsidian=./eval/claude-probe/obsidian-stub.ts \
 *     --outfile=eval/claude-probe/run.cjs
 *   node eval/claude-probe/run.cjs
 */
import { probeClaudeBinary } from "../../src/claude-cli-client";

let pass = 0, fail = 0;
const failures: string[] = [];
function check(name: string, cond: boolean, detail = ""): void {
  if (cond) { pass++; console.log(`  PASS  ${name}`); }
  else { fail++; failures.push(name); console.log(`  FAIL  ${name}${detail ? `\n        -> ${detail}` : ""}`); }
}
async function resolves(fn: () => Promise<unknown>): Promise<boolean> {
  try { await fn(); return true; } catch { return false; }
}
async function rejects(fn: () => Promise<unknown>): Promise<boolean> {
  try { await fn(); return false; } catch { return true; }
}

async function main(): Promise<void> {
  console.log("=== probeClaudeBinary ===");
  // process.execPath is an absolute path to the Node binary, which supports
  // `--version` and exits 0 — stands in for a healthy Claude CLI binary.
  check("valid binary resolves", await resolves(() => probeClaudeBinary(process.execPath)));
  check("missing binary rejects", await rejects(() => probeClaudeBinary("/nonexistent/zzz-claude-binary")));
  check("relative path rejects", await rejects(() => probeClaudeBinary("relative/claude")));
  check("empty path rejects", await rejects(() => probeClaudeBinary("")));
  check("traversal path rejects", await rejects(() => probeClaudeBinary("/opt/../etc/claude")));

  console.log(`\n${pass} passed, ${fail} failed`);
  if (fail > 0) { console.log("FAILURES:\n  " + failures.join("\n  ")); process.exit(1); }
}
main();
```

- [ ] **Step 3: Implement `probeClaudeBinary`**

In `src/claude-cli-client.ts`, immediately after the `validateIclaudePath` function (which ends at the line `}` before `export class ClaudeCliClient`), add:
```typescript
/**
 * Verify the configured Claude CLI binary exists and runs by spawning it with
 * `--version` and checking the exit code. Replaces an earlier fs.access(X_OK)
 * check so the plugin no longer imports the Node `fs` module. Desktop-only;
 * child_process is loaded lazily so the Node builtin never executes on mobile.
 */
export async function probeClaudeBinary(iclaudePath: string): Promise<void> {
  validateIclaudePath(iclaudePath);
  if (!Platform.isDesktopApp) throw new Error("Claude CLI backend is desktop-only");
  const { spawn } = await import("node:child_process");
  await new Promise<void>((resolve, reject) => {
    const child = spawn(iclaudePath, ["--version"], { stdio: ["ignore", "ignore", "pipe"] });
    let stderr = "";
    const timer = setTimeout(() => { child.kill("SIGTERM"); reject(new Error("timeout")); }, 5000);
    child.stderr?.on("data", (d: Buffer) => { stderr += d.toString("utf8"); });
    child.on("error", (err: Error) => { clearTimeout(timer); reject(err); });
    child.on("close", (code: number | null) => {
      clearTimeout(timer);
      if (code === 0) resolve();
      else reject(new Error(stderr.trim() || `exit ${code}`));
    });
  });
}
```
Note: `setTimeout` / `clearTimeout` are used unprefixed (not `window.setTimeout`) so the function runs both in the Obsidian renderer and headlessly under Node in the eval.
