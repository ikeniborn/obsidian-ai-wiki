# Agent Base Contract — Design Spec

Date: 2026-05-04

## Problem

Правила поведения LLM-агента (достоверность, формат, минимализм) сейчас либо разбросаны по фазовым промтам, либо отсутствуют. Нет единого зафиксированного контракта, который применяется ко всем операциям.

## Goal

Создать единый базовый системный промт (`prompts/base.md`), который:
- встраивается при сборке в `main.js` (через существующий esbuild text loader)
- инжектируется первым в системный промт каждой операции, до фазового промта и до `## Уточнение`
- содержит только универсальные правила (специфика остаётся в фазовых промтах)

## Out of scope

- Изменение фазовых промтов
- Изменение механизма `settings.systemPrompt`
- Новые настройки или UI

## Architecture

### Порядок системного промта (итоговый)

```
[prompts/base.md]          ← базовый контракт (hardcoded)
[prompts/<phase>.md]       ← фазовый промт (уже существует)
## Уточнение               ← settings.systemPrompt (если задан пользователем)
```

### Точка инжекции

`src/phases/llm-utils.ts` — функция `buildChatParams()`.

Импорт при сборке:
```typescript
import baseContract from "../../prompts/base.md";
```

Инжекция: `baseContract` prepend-ится к первому системному сообщению перед фазовым промтом. Не параметр функции — константа модуля, всегда одна и та же.

### Механизм сборки

esbuild уже настроен: `loader: { ".md": "text" }`. Все `.md`-файлы в `prompts/` импортируются как строки статически. Никакого disk read в runtime.

## Content of `prompts/base.md`

```markdown
Ты — wiki-агент. Следуй этим правилам независимо от операции.

## Достоверность
Отвечай строго на основе предоставленного контекста.
Не выдумывай факты, которых нет в источнике.
Если контекста недостаточно — скажи об этом прямо.

## Формат
Возвращай ровно то, что запрошено.
Если ожидается JSON — только валидный JSON, без пояснений вокруг.
Если ожидается текст — без служебных меток и технических артефактов.

## Минимализм
Не добавляй то, о чём не просили.
Не комментируй собственные действия, если это не часть задачи.
```

## Changes Required

| Файл | Изменение |
|------|-----------|
| `prompts/base.md` | Создать (новый файл) |
| `src/phases/llm-utils.ts` | Импортировать `base.md`, prepend в `buildChatParams` |

## Testing

- Unit-тест в `tests/llm-utils.test.ts`: `buildChatParams` с системным сообщением → базовый контракт идёт первым
- Визуальная проверка: запустить любую операцию в dev-режиме, убедиться в JSONL-логе что `systemPrompt` содержит базовый блок
