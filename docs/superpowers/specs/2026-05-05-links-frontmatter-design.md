# Дизайн: перенос исходящих ссылок в Frontmatter

**Дата:** 2026-05-05  
**Статус:** approved

## Проблема

Раздел `## Связанные концепции` обязателен по схеме и генерируется LLM в каждой wiki-статье. Он воспринимается как технический артефакт, засоряющий видимый контент, а не несёт смысловой нагрузки для читателя.

## Решение

Перенести WikiLinks на связанные страницы из тела статьи в YAML-frontmatter под поле `links`. Obsidian 1.4+ распознаёт `[[...]]`-синтаксис внутри YAML-значений и включает такие ссылки в Graph View и Backlinks нативно.

## Затронутые файлы

| Файл | Изменение |
|---|---|
| `templates/_schema.md` | Убрать п.5 из обязательной структуры страницы; добавить `links` в таблицу Frontmatter |
| `prompts/ingest.md` | Добавить `links: []` в пример JSON-ответа LLM |
| `src/phases/lint.ts` | Без изменений |
| `src/phases/ingest.ts` | Без изменений |

## Формат поля `links`

```yaml
links:
  - "[[concept-a]]"
  - "[[concept-b]]"
```

Пустой массив допустим: `links: []`.

## Совместимость с lint

`checkStructure` в `lint.ts` применяет regex `/\[\[([^\]]+)\]\]/g` к полному raw-content страницы, включая текст YAML-frontmatter. Ссылки в `links` будут проверяться на наличие целевой страницы автоматически — поведение не меняется.

## Обратная совместимость

Существующие статьи с разделом `## Связанные концепции` не мигрируются. Новые статьи генерируются по обновлённой схеме. Lint не сломается для старых статей — секция 5 остаётся валидным markdown-контентом.

## Требования к Obsidian

Obsidian ≥ 1.4.0 (сентябрь 2023). Для более старых версий frontmatter-ссылки не попадут в граф.
