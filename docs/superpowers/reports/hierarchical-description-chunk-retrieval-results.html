Single-domain

1. Чтение доменаЧитаются страницы домена и description из frontmatter через collectDescriptions.
2. Seed-статьиEmbedding/hybrid сравнивает query только с summary vectors; при сбое или слабом сигнале работает Jaccard fallback.
3. Graph poolbfsExpandRanked расширяет seeds по wiki graph; floor/fusion остаются article-level решениями.
4. Итоговые article idsБерётся graph candidate pool или fused subset; это только пул для detailed retrieval.
5. Clean chunksselectRelevantChunks ранжирует section chunks внутри пула; section input = heading + body, без description.
6. Контекст promptrenderContextChunks отдаёт только article id, section heading и chunk body; index scoped только к final selected ids.
7. Защита ответаanswerFromContext валидирует WikiLinks только против final selected ids.

Cross-domain

1. Кандидаты по доменамДля каждого домена выполняется тот же seed → graph pool этап.
2. Merge/fusemergeCandidates объединяет candidate pools и применяет fusion/cap на article ids.
3. Chunk rerankФинальный chunk search идёт только внутри capped merged article pool.
4. Chunk-only contextPrompt содержит selected chunks; cross-domain index строится только по finalChunkIds.
5. Scoped linksAvailable WikiLinks, validation, found_pages и found_chunks основаны на selected chunk article ids.
