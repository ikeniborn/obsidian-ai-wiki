2. Схемы решения и зависимостей

Схема решения (компоненты и поток управления)

Поток R2b: runLint → (actualize) → cleanup считает файлы в !Wiki/<folder>/<subfolder>/ по живой карте pages → для count 0: rmdir + domain_updated{entity_types: survivors} → controller сохраняет → view.refreshDomains().

Граф зависимостей (A→B = A зависит от/использует B)

Циклов нет (DAG). R1 и R2a независимы; R2b опирается на rmdir и пайплайн domain_updated.
