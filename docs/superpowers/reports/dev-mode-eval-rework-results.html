Intent — цель и рамки

Заменить LLM-judge авто-скоринг dev-mode на человеческие 👍/👎-метки качества, сохранять их как gold-датасет и переиспользовать в scripts/eval.ts (сравнение вариантов) и scripts/dspy (оптимизация промптов).

Desired outcomes

  • Per-run запись в eval.jsonl на финише с rating: null; клик 👍/👎 правит её по runId (флип, без дублей).
  • query/chat: одна 👍/👎 на ответе (desktop + mobile).
  • format: две оси — formatting (всегда) и recognition (когда был vision), бакеты vision on/off.
  • Per-run телеметрия: llmErrors[] + ruleFirings{} по 4 группам детерминированных правил.
  • eval.ts: answer/format/recognition-метрики + telemetry-report per prompt.
  • dspy: оптимизация из 👍+👎, включая format.md + vision-*.md, без регресса 👍-кейсов.

Constraints (ключевые)

  • Hard: текст промптов только в prompts/*.md; счёт правил — типизированным сигналом (без парсинга info_text); телеметрия без лишних LLM-вызовов; mobile — node builtins lazy + desktop-guard.
  • Steering: запись/телеметрия/кнопки только при devMode.enabled; non-dev путь не трогать; PR-only в master.

Spec — резюме решения

Каждый dev-mode ран query/chat/format пишет одну запись в eval.jsonl (каталог плагина) с provenance + телеметрией; клик 👍/👎 правит запись по runId. eval.ts + dspy читают датасет. LLM-judge и ретривал-харнес удаляются.

Требования по секциям

§Требование
1Всё под devMode.enabled; решения: rule_fired event, content-hash promptVersion, бинарная dspy-метрика + 👍-guard; storage в plugin-dir (не синк, per-device).
2Единый runId = String(startedAt) из controller → в запись + vision-temp + view (finish/format_preview).
3Пути в plugin-dir (runtime-resolved); авто-миграция волтовых _dev/_agent.jsonl при обновлении (идемпотентно); легаси-строки без rating пропускаются.
4writeEvalRecord / updateEvalRating(runId,axis,rating); схема query/chat и format (две оси); promptVersion=hash8(tpl), visionPromptVersion = hash8 от per-template хешей; retrievalConfig-снимок.
5Новый rule_fired{ruleId,count} event; AgentRunner копит llmErrors[]+ruleFirings{} in-memory, флашит в запись. 4 группы правил; счёт — структурно.
6Кнопки 👍/👎: query/chat в finish(); format — formatting всегда + recognition при visionCount>0; desktop+mobile; i18n en/ru/es.
7eval.ts читает eval.jsonl → answer/format/recognition-метрики (бакеты vision) + telemetry-report per prompt.
8dspy: rating вместо score; метрика 1/0; 👍-held-out guard (reject если mean < baseline); трейнсеты query/chat + format(vision)/recognition.
9Удаления: evaluator.ts/evaluator.md, evaluatorModel, eval_result; ретривал-харнес eval-*.ts+scripts/eval/; rename файлов.
10Health: 0 лишних LLM-вызовов; 1 append/ран + 1 RMW/клик; non-dev не тронут; mobile build ок; lint чисто.
11–13Фазировка для writing-plans; docs/wiki через iwiki-ingest; trust-приоритет + human-checkpoints.
