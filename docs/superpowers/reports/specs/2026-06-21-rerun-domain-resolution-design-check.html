1. Резюме спецификации

Спецификация устраняет баг повторного запуска (re-run) из истории: запрос уходил на неверный домен (domains[0]), потому что domainId «прогонялся» через DOM <select> и терялся. Решение — сделать сохранённый RunHistoryEntry.domainId источником истины и валидировать его.

Требования по секциям

СекцияТребование
ScopeЧинить только путь re-run-из-истории; использовать сохранённый domainId напрямую, не через DOM.
Scope / Out of scopeНе трогать fallback domains[0] в src/phases/query.ts, путь command-palette, fallback'и detectDomain/DomainModal.
Design · BehaviorПри клике ↺: домен берётся из записи истории, а не из select.
Design · BehaviorЕсли domainId совпадает с существующим доменом — запуск с этим id, переданным напрямую в controller.query.
Design · BehaviorЕсли domainId отсутствует или не найден — показать Notice и не запускать запрос.
Design · Code changesНовый чистый модуль src/rerun-domain.ts с функцией resolveRerunDomain(entry, domains) (тестируемой в изоляции).
Design · Code changesПереписать re-run handler в src/view.ts: loadDomains() → resolveRerunDomain → Notice либо controller.query(..., r.domainId).
Design · i18nДобавить ключ(и) сообщения об ошибке во все три локали en/ru/es (иначе падает tsc).
TestingUnit (headless, esbuild --alias:obsidian=stub) на 3 случая resolveRerunDomain; ручная проверка mobile+desktop.
DocsОбновить страницы docs/wiki (iwiki) по query / history re-run / domain resolution.

Success Criteria (критерии приёмки)

2. Схемы решения и зависимостей

2.1 Схема решения — поток управления re-run

Сравнение «было» (DOM как источник истины — баг) и «стало» (запись истории как источник истины с валидацией).

БЫЛО (баг) ↺ click → domainSelect.value = it.domainId → dispatch change → submitQuery() → читает select.value обратно. Если option нет → "" → undefined → domains[0] (неверный домен).
СТАЛО (фикс) ↺ click → loadDomains() → resolveRerunDomain(it, domains) → либо Notice (stop), либо controller.query(q, r.domainId) напрямую.
↺ Re-run clicksrc/view.ts rerunBtn listener
▼
controller.loadDomains()авторитетный список доменов, без гонки select
▼
resolveRerunDomain(it, domains)src/rerun-domain.ts — чистая функция
▼
!ok → missing / not-foundnew Notice(...) → return (запрос не идёт)
ok → domainIdsync dropdown → controller.query(it.args[0], r.domainId)
