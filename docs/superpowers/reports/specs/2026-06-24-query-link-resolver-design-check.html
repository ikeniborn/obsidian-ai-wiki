Требования по секциям

СекцияТребование
Goals.1Структурированный вывод (zod) — контракт ответа модели.
Goals.2Убрать дополнительный проход: модель сразу выдаёт валидные ссылки, частота per-query rewrite LLM-вызова → к нулю.
Goals.3Ноль битых ссылок в финальном ответе (нет остаточных *(not in wiki)*).
Goals.4Диагностируемость: в логе видно, что было сломано и как разрешено.
Components.1link-resolver.ts (~60 строк), детерминированный, без LLM. Нормализация (lowercase, срез wiki_<domain>_, удаление пробелов/дефисов, извлечение id-фрагмента), затем группировка совпадений по сущности. Source-заметка и её wiki_*-страница = одна сущность. → 1 сущность: resolved (канонический стем, предпочтение wiki_*); ≥2 разных сущности: ambiguous; 0: unresolved. Предпочтение wiki_* применяется после группировки и не схлопывает разные сущности.
Components.2QueryAnswerSchema { reasoning, answer_markdown, citations }, superRefine: каждый citation в knownStems. Только fallback.
Components.3Оркестрация query.ts: draft → extractAnswerLinks → findBrokenLinks → link-resolver (0 LLM) → при остатке и wikiLinkValidationRetries>0: parseWithRetry(QueryAnswerSchema) → assistant_replace.
Components.4query-link-validator.ts: сохранить extract/find/annotate; rewriteWithValidLinks понижен до fallback.
Prompt hardening{{available_links_block}} в prompts/query.md; query.ts рендерит нумерованный список валидных стемов (wiki_* preferred).
DiagnosticsСтруктурированный preview FixingLinks: resolved N (det) / llm-fixed N / stripped N / annotated N.
Hybrid streamingreasoning+draft стримятся live; после резолвера assistant_replace; fallback parseWithRetry без стриминга ответа.
SettingsПереиспользовать wikiLinkValidationRetries (default 3); 0 = резолвер+strip, без LLM-fallback. Новых настроек нет.

Success Criteria (Verification)

#Критерий приёмки
1npm run build успешен; eslint чисто.
2Повторный query на rtk-task: в логе resolved N (det), нет llm-fixed, финал без *(not in wiki)*.
3Негативный кейс: ссылка на несуществующую страницу → stripped/annotated, никогда сырой битой ссылки.
4 обновлёнКейс неоднозначности: битый стем, чей фрагмент матчит две разные сущности (напр. 88393 в wiki_x_88393 И wiki_y_188393) → ambiguous → annotated. Обратно: source-заметка + её wiki_*-страница (одна сущность) обязаны resolve к wiki_*-стему, не ambiguous.
5iwiki-ingest изменённых источников + iwiki-lint.
