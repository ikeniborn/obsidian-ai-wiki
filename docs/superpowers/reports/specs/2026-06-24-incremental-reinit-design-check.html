Блок 1 — Резюме спецификации

Требования по секциям

СекцияТребование
ObjectiveДобавить инкрементальный режим reinit; на старте пользователь выбирает Full или Incremental; Incremental переингестит только изменённые источники после подтверждения списка.
Component 1
vault-tools.ts		Новый accessor mtime — VaultAdapter.stat?(path) и VaultTools.mtime(vaultPath); фазовый код читает mtime только через VaultTools, не импортирует obsidian.
Component 2
incremental-sources.ts (новый, pure)		computeChangedSources: associated пуст → include; mtime недоступен → include (trust bias); mtime(S) > min(mtime associated pages) → include; иначе skip. Строгое > и min — load-bearing.
Component 2awiki_sources используется только для структурного отображения source↔page, не как сигнал свежести; свежесть — только по mtime.
Component 3
modals.ts		Новый ReinitModeModal: кнопки Full / Incremental(N) / Cancel; Incremental disabled при N=0; список изменённых basenames — первые 20, «+K more» когда список длиннее 20.
Component 4
view.ts / controller.ts		computeIncrementalPlan(domainId) → {changed, totalSources, wikiFileCount}; init(...) получает incremental?:boolean → добавляет --incremental; runReinit открывает модал и диспетчеризует.
Component 5
phases/init.ts		Новая ветка --incremental в runInit (до --force/--sources, взаимоисключающая с --force); без wipe/bootstrap/reset; цикл runIngest по changed; не рефильтрует по analyzed_sources; новые источники дописываются.
Component 6
phases/ingest.ts
HUMAN CHECKPOINT		A2 reorder: писать source до страниц, чтобы wiki-страницы были не старше источника (инвариант mtime); existingArticles через parseWikiArticlesFromFm; backlinks из planned-набора; trade-offs (page write fail, dedup-merge) self-healing.
Edge casesN=0 disabled; новый источник без страницы → include; страницы удалены вручную → include; shared page → min+strict> держит неизменённый источник неотмеченным; barren source (0 страниц) → всегда include (известное ограничение); mobile — desktop-only.
Verificationunit (computeChangedSources out-of-vault), integration (node fs, A2 write order, mtime(page)≥mtime(source)), manual e2e (homelab LLM).
i18nНовые ключи ru/en/es для ReinitModeModal.
Out of scopeПерсистентные timestamps; изменения ingest помимо A2; умная переингест barren; mobile.

Success criteria (из intent)

#Критерий
1На старте reinit пользователь выбирает Full или Incremental.
2Incremental показывает диалог только с изменёнными источниками + confirm; кнопка disabled при пустом списке.
3На confirm ingest идёт последовательно только по списку; нетронутые не переингестятся.
4Новый источник (в source_paths, без страницы) появляется и ингестится.
5Full reinit (--force) вывод не меняется (страницы, _index.md, domain entry идентичны).
6Список изменённых — строгое подмножество всех источников.
