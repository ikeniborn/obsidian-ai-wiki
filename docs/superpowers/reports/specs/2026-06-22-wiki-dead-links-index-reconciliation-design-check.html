1. Резюме спецификации

Спека описывает два детерминированных исправления плагина Obsidian AI-wiki: удаление мёртвых ссылок и двунаправленную реконсиляцию _index.md с fallback-аннотацией. Корень обеих проблем — инварианты были только промпт-инструкциями к LLM, а не принудительно проверяемыми правилами.

Требования (по секциям)

СекцияТребование
§Approach · Component 1stripDeadLinks(content, knownStems) в src/wiki-link-validator.ts: удаляет из тела каждый [[stem]], где stem ∉ knownStems; чистит артефакты (двойные пробелы, висячие ,/./;, trailing); пустые ячейки таблиц оставляет; пере-выводит wiki_outgoing_links из очищенного тела через setFmLinks; запускается безусловно внутри fixWikiLinks.
§Approach · Component 2deriveFallbackAnnotation(content, entityType?) в src/wiki-index.ts: вывод "<H1> — <первое непустое предложение>. Type: <entityType ?? 'general'>", одна строка, обрезка ~600–800 символов; entityType из подпапки/frontmatter, fallback general.
§Approach · Component 3reconcileIndex(indexContent, pageFiles, getAnnotation) → {adds, removes} в src/wiki-index.ts. Add: файл-страница проходит GENERIC_WIKI_STEM_REGEX, не _*, pid отсутствует в индексе (аннотация = аннотация прогона или deriveFallbackAnnotation; секция через deriveSection). Remove: pid в индексе без файла на диске. Применяется через upsertIndexAnnotation / removeIndexAnnotation.
§Integration · ingest.tsstripDeadLinks встроен в шаг fixWikiLinks (~line 313); в write-loop всегда индексировать page.annotation ?? deriveFallbackAnnotation (заменяет skip на if(page.annotation) @line 408); после write-loop — reconcileIndex по nonMetaPaths.
§Integration · lint.tsВ always-on post-loop блоке (вне if(useLlm), ~line 535): stripDeadLinks по каждому телу с vault-wide knownStems (@line 219) + ресинк frontmatter; reconcileIndex по страницам домена — двунаправленно, с LLM и без.
§Error handlingЗаписи в индекс не критичны (try/catch /* non-critical */); stripDeadLinks — чистая работа со строкой, на malformed-странице работает по всему контенту как по телу; reconcileIndex никогда не удаляет файлы страниц, только правит _index.md.
§Decisions(1) dead link → удалить токен целиком с очисткой; (2) индекс — и предотвращение, и лечение; (3) реконсиляция двунаправленная; (4) fallback annotation = H1 + первое предложение + Type, детерминированно без LLM; (5) всё работает без LLM.

Success Criteria (§Verification criteria)

№Критерий приёмки
1После ingest или lint (LLM on/off) на rtk-task grep не находит [[wiki_rtk-task_ch_mete_s3_ddrdrg]] нигде (тело/frontmatter).
2Diff индекс ↔ диск пустой: у каждого файла-страницы есть запись в индексе; нет записей без файла.
3ch_mete_s3_ddrd (и ещё 10 страниц) появляются в _index.md и после refreshCache возвращаются по релевантному запросу.
4Новые eval-тесты проходят; нет новых tsc-ошибок в затронутых файлах (baseline не чист — gate только на новые ошибки).
