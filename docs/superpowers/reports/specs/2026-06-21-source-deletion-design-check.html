1. Резюме спецификации

Требования (по секциям)

#СекцияТребование
R1Architecture · Pure plannerЧистый модуль src/source-deletion.ts: computeDeletionPlan → {toDelete, toRebuild, remainingSources}, isSourceFile. Стем-матчинг точный (не includes). Единый источник истины для preview и фазы.
R2Architecture · Execution phaseНовая WikiOperation "delete" + фаза runDelete (src/phases/delete.ts). 8 шагов: config-update → wipe → yield* runIngest по remaining → delete sole → backlinks → условное удаление source → cache invalidate → result.
R3Architecture · UIdeleteBtn рядом с formatBtn (desktop+mobile), отдельная иконка. Общий гейт isSourceFile для обоих. DeleteSourceModal с N/M + списками + предупреждением о безвозвратности.
R4Error handlingСбой пересборки: collect+continue, source сохраняется (F-002). Невалидный путь: validateArticlePath отклоняет. План непосчитуем: halt. Abort между шагами. Mobile/оба backend.
R5TestingOut-of-vault eval eval/source-deletion/run.ts на computeDeletionPlan (7 кейсов) + ручной прогон на реальном vault.

Success Criteria (Done when)

Клик Delete → корректные счётчики/списки в модали; по подтверждению source-файл перманентно удалён и убран из домена; каждая sole-source страница удалена; каждая multi-source пересобрана на оставшихся; lint/check — ноль осиротевших/битых ссылок/stale-артефактов; сбои пересборки в финальном отчёте.
