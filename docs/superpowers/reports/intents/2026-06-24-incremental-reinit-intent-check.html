1 · Резюме требований

Objective

Re-init домена умеет только полный rebuild (--force стирает всю вики-папку и переингестит все источники) — дорого, когда изменилось несколько источников. Добавить инкрементальный режим: при старте reinit выбор Полный / Инкрементальный; инкремент переингестит только изменённые с момента последнего отражения в вики источники после подтверждения списка.

Desired Outcomes

IDНаблюдаемый результат
DO1При старте reinit пользователь выбирает режим — Полный или Инкрементальный.
DO2Инкремент открывает диалог со списком только источников, у кого mtime файла-источника новее mtime связанной вики-страницы, + кнопка подтверждения.
DO3По подтверждению ingest идёт последовательно только по списку; нетронутые источники не переингестятся.
DO4Кнопка «Инкрементальный» неактивна, когда переингеститься нечему.
DO5Новый источник (в source_paths, но без вики-страницы) попадает в список и инжестится.

Health Metrics

IDНе должно деградировать
HM1Полный reinit (--force) — байт-в-байт без изменений.
HM2Инкремент = строгое подмножество; неизменённые источники никогда не переингестятся (критично: при mtime-бампе фича вырождается в полный прогон).
HM3Существующий ingest-конвейер на источник без изменений (entity extraction, stem mask, link & index hygiene).
HM4Frontmatter источников не трогается новой логикой выбора (только чтение mtime).

5 · Stop Rules

  • Done when: на волте с N источниками, где правлено K → инкремент-диалог показывает ровно K (+новые), ingest идёт только по ним, полный reinit на том же волте без изменений.
  • Halt if: полный reinit (--force) меняет наблюдаемое поведение.
  • Escalate if: сравнение mtime не может надёжно исключить неизменённые источники (инвариант недостижим без хранения state).
