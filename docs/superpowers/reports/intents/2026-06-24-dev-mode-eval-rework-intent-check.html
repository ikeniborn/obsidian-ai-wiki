1. Резюме требований

Objective

Заменить недоверенного LLM-судью (devMode.evaluatorModel → runEvaluator → prompts/evaluator.md → eval в _dev.jsonl) на человеческие 👍/👎 метки качества как gold-датасет и переиспользовать его для (a) сравнения вариантов через scripts/eval.ts и (b) улучшения промтов через scripts/dspy. Скоуп — две операции: query/chat (одна оценка ответа) и format (две оси — форматирование всегда + распознавание когда был vision; бакеты vision on/off). Каждый dev-прогон пишет harness-телеметрию (LLM-ошибки + счётчик срабатываний детерминированных правил) для решения «править промт vs править харнес».

Desired Outcomes

IDРезультат (наблюдаемый)

2d. Stop Rules — Done when

  1. dev-режим: 👍/👎 под ответом query/chat на desktop+mobile; клик обновляет запись eval.jsonl (runId, question, found_pages[], answer, rating, promptVersion, model, retrievalConfig, llmErrors[], ruleFirings{}).
  2. dev-режим: превью format показывает formatting 👍/👎 всегда и recognition 👍/👎 при vision (desktop+mobile); клик обновляет format-запись.
  3. Каждый dev-прогон добавляет ровно одну per-run запись с llmErrors[] и ruleFirings{} по 4 группам, структурно (без парсинга info_text).
  4. non-dev query/chat/format/vision не изменён; mobile-сборка собирается и работает.
  5. npm run lint чисто; нет новых tsc-ошибок в тронутых файлах.
  6. scripts/eval.ts выдаёт answer/format/recognition метрики (бакеты vision on/off) + агрегатный telemetry-отчёт.
  7. scripts/dspy потребляет 👍+👎 вкл. format/vision; оптимизирует prompts/+templates/ (вкл. format.md+vision-*.md) без регресса 👍.
  8. LLM-судья (evaluator.*, devMode.evaluatorModel, eval_result) и retrieval-eval harness удалены.
Halt: если удаление evaluator/retrieval harness, rename файла или инструментирование телеметрии изменит non-dev путь или сломает mobile-сборку. Escalate: если метку (и её телеметрию) нельзя достоверно сматчить на единственную запись prompt/model/config; или правило не может отдать счётчик без парсинга текста.
