1. Резюме требований

Objective

Сегодня dev mode авто-оценивает каждый прогон отдельным LLM-судьёй (devMode.evaluatorModel → runEvaluator → prompts/evaluator.md → поле eval в !Wiki/_config/_dev.jsonl). Сигнал судьи недостаточно надёжен для работы над промптами. Заменить его на собранные человеком метки качества ответов query/chat, сохраняемые как gold-датасет, и переиспользовать датасет для (a) сравнения вариантов через scripts/eval.ts и (b) улучшения промптов на плохих ответах через scripts/dspy. Делается сейчас, чтобы накопленные 👍/👎 стали ground truth до дальнейшего тюнинга промптов.

Desired Outcomes

#Наблюдаемый результат
DO1В dev mode под ответом query/chat появляются две кнопки 👍/👎 на desktop и mobile; клик пишет/обновляет одну запись в !Wiki/_config/eval.jsonl.
DO2Каждая запись содержит минимум {question, found_pages[], answer, rating} плюс достаточно провенанса для привязки метки к конкретному артефакту: версия промпта/шаблона, модель, retrieval-config. Схему можно расширять под богаче сигнал для eval + dspy.
DO3Повторный клик 👍/👎 обновляет rating существующей записи (без дублей строк).
DO4scripts/eval.ts читает eval.jsonl и выдаёт метрику качества агента по размеченным примерам.
DO5scripts/dspy потребляет размеченный набор (👍 и 👎) и оптимизирует промпты из prompts/ + templates/, используя 👍-кейсы как защиту от регрессии хороших ответов.

Health Metrics

#Метрика
HM1Нормальное (non-dev) поведение query/chat и производительность не меняются — запись и кнопки gated на devMode.enabled.
HM2Mobile-сборка не падает (node builtins остаются lazy + desktop-guarded; кнопки рендерятся на mobile).
HM3npm run lint чистый; нет новых tsc-ошибок в затронутых файлах.

Strategic Context

Interacts withquery/chat phase (источник данных — graph_stats found pages + result answer), src/view.ts (рендер ответа + кнопок 👍/👎), settings.ts (флаг devMode), !Wiki/_config/eval.jsonl (общий, синкается Obsidian между desktop/mobile — non-critical), scripts/eval.ts (Node/tsx harness), scripts/dspy (Python, отдельный LLM API для оптимизации промптов).
Priority trade-offtrust — точность сигнала метки и корректная атрибуция метки к промпту/модели/конфигу, породившему ответ.

Constraints

#ТипОграничение
S1SteeringЗапись + кнопки активны только при devMode.enabled; non-dev путь нетронут.
S2Steeringeval.jsonl — append-only JSONL; повторный клик 👍/👎 редактирует rating записи, а не добавляет дубль.
S3SteeringКаждая запись фиксирует версию промпта/шаблона, модель и retrieval-config, чтобы 👎 однозначно мапился на один артефакт (требование dspy).
S4SteeringИ eval.ts, и dspy читают промпты из prompts/ + templates/.
H1HardMobile: никаких node builtins в bundle плагина (lazy + desktop-guard); 👍/👎 рендерятся и на mobile.
H2HardТекст промптов остаётся в prompts/*.md (никаких хардкод-строк промптов в .ts).
H3HardУдалить: src/phases/evaluator.ts, prompts/evaluator.md, devMode.evaluatorModel, событие eval_result, и обвязку runEvaluator в AgentRunner.
H4HardВывести из эксплуатации retrieval-eval harness: scripts/eval-*.ts + scripts/eval/ gold-каталог. Новый scripts/eval.ts читает eval.jsonl (качество ответа заменяет retrieval Recall@k/MRR).
H5HardПереименовать лог-файл _dev.jsonl → eval.jsonl (обновить src/wiki-path.ts + src/storage-migration.ts).
H6HardНе коммитить напрямую в master; работать в ветке dev/*, мерж через PR.

Stop Rules — условия завершения (Done when)

  1. В dev mode 👍/👎 видны под ответом query/chat на desktop и mobile; клик пишет/обновляет запись eval.jsonl {question, found_pages[], answer, rating, версия промпта/шаблона, модель, retrieval config}.
  2. Non-dev query/chat не изменён; mobile-сборка собирается и запускается.
  3. npm run lint чистый; нет новых tsc-ошибок в затронутых файлах.
  4. scripts/eval.ts выдаёт метрику качества агента из eval.jsonl.
  5. scripts/dspy потребляет размеченный набор (👍 + 👎) и оптимизирует промпты из prompts/ + templates/ без регрессии 👍-кейсов.
  6. LLM-судья (evaluator.*, devMode.evaluatorModel, eval_result) и retrieval-eval harness удалены.
Halt-if: удаление evaluator/retrieval harness или переименование файла изменило бы non-dev путь query/chat или сломало mobile-сборку.
Escalate-if: надёжная атрибуция метки к одной записи промпт/модель/конфиг недостижима на доступных run-данных.
