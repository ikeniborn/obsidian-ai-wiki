1. Резюме спецификации

Раздел / требованиеСуть
ScopeOKF применяется только к генерируемым wiki-страницам; source-заметки остаются вне OKF.
Wiki-frontmatterКонвергенция к OKF-именам, снятие префикса wiki_.
type= подкаталог-тип (!Wiki/<domain>/<type>/…); entities→concept; layout не меняется.
description / titledescription на диске (из ingest-annotation, 1–2 предл.); title выводится при экспорте.
Переименованияwiki_sources→resource, wiki_updated→timestamp, wiki_status→status, wiki_outgoing_links→outgoing_links, wiki_external_links→external_links; wiki_type удалён.
Source-заметкиОставить wiki_-префикс и wiki_articles; убрать wiki_added+wiki_updated (свежесть = хеш тела, не страдает).
МиграцияWiki-страницы: активная one-shot идемпотентная авто-миграция в цепи main.ts (hard cutover). Source-даты: ленивая зачистка на следующем ingest/format.
ExportOffline TS-порт iwiki export.py; near-passthrough frontmatter + вывод title + kebab-теги (a/b→a-b) + [[wikilinks]]→[md](links) + генерация index.md/log.md; dest-модалка, дефолт вне vault.
Success CriteriaWiki-страницы OKF-native на диске; export даёт conformant bundle offline; миграция идемпотентна (2-й проход без изменений); свежесть источников не нарушена.
