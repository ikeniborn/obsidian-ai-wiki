1. Резюме спецификации — требования по разделам

РазделIDТребование (одно на строку)
Decisions · ScopeR1Все 8 LLM-операций (ingest, query, lint, lint-chat, chat, init, format, delete) получают rating-кнопки + запись eval.jsonl.
R2Гранулярность — одна запись + набор рейтингов на runId (run-level), без per-item записей; форма несёт список обработанных items как provenance.
Design §1 · Схема рейтинговR3EvalRecord.ratings: Record<string, Rating> заменяет скалярные rating/recognitionRating; ключ появляется только после клика по оси; инициализация {}.
R4OPERATION_AXES: Record<WikiOperation, AxisDef[]> — единый источник осей, потребляемый view и позже eval.ts/dspy; RatingAxis расширяется до string.
R5recognition несёт gate:"vision" — рендерится только при visionCount > 0 (format-only).
Design §2 · ProvenanceR6EvalMetaFields расширяется опциональными полями (created_pages, updated_pages, source_paths, files_processed, domain, articles, instruction, deleted_source, rebuilt_pages); существующие поля без изменений.
R7Каждая фаза эмитит eval_meta в конце run из уже посчитанных данных — ноль доп. LLM-вызовов, без mid-run I/O; accumulator в agent-runner мёржит.
Design §3 · UI / view.tsR8Defect 1 (reset): private ratingSection; в reset() до пере-вооружения run — ratingSection?.remove() и formatPreviewSection?.remove() с обнулением.
R9Defect 2 (render): в finish() для каждой операции кроме format при status==="done" создаётся ratingSection внутри resultSection и цикл по OPERATION_AXES[op]; entry.id = canonical runId, привязка к отображаемому результату.
R10format держит оси в renderFormatPreview (formatting всегда, recognition при visionCount>0), читая из OPERATION_AXES["format"]; QC_OPS/hard-coded answer-ветка заменена registry-driven циклом.
Design §4 · WriterR11agent-runner.ts: конструкция EvalRecord убирает rating:null/recognitionRating, ставит ratings:{}; остальное (телеметрия, merge, gating) без изменений.
Design §5 · ControllerR12rateRun(runId, axis:string, rating) — тип axis расширяется до string, форвардит в updateEvalRating; без логики.
Design §6 · i18nR13Добавить метки осей en/ru/es: retrieval, page, links, coverage, fix, rebuild; существующие ratingAnswer/Formatting/Recognition остаются.
