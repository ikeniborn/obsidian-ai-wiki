Требования по секциям

СекцияТребование
Part 1 — Stats spacingВ src/styles.css (.ai-wiki-cross-stats, ~стр. 390) добавить margin-top: 10px, чтобы блок статистики отделялся от заголовка «Results». Других изменений нет.
Part 2 — «Selected for LLM» transparencyВ query.ts расширить query_stats полями seedCount / graphCount (только single-domain): seedCount = seeds.length, graphCount = contextPages.length − seedCount. В view.ts (renderQueryStats, ~стр. 960) выводить строку выбора как «7 (5 vector + 2 graph)». Cross-domain (Ask Wiki) статистика структурно иная и разбивку НЕ получает. После работы floor (Part 3) graphCount естественно уменьшается — разбивка служит видимым доказательством срабатывания floor.
Part 3 — Relevance-floor pruning (core)Новый чистый помощник src/retrieval-prune.ts: pruneByRelevance(expandedDense, denseRef, ratio) → Set<pid> — пиды-«оставить» (expandedDense[pid] ≥ ratio · denseRef); seeds не передаются (всегда сохраняются). Подключение в retrieveDomainCandidates сразу после bfsExpandRanked, до candidateIds. denseRef = denseMax (per-domain raw cosine лучшего seed, query.ts:100). Per-page cosine для graph-страниц переиспользуется из seed dense pass (selectEmbeddingScoredDiag → расширить SeedDiag картой denseByPid) — ноль дополнительных embedding-вызовов; bfsExpandRanked не меняется. Ordering (fuseVectorGraph/seeds-first) и membership разделены — floor меняет только membership. Матрица применимости + observability (graph_stats: floorApplied/floorRef/prunedCount/floorSkippedReason). Floor живёт per-domain в retrieveDomainCandidates → Ask Domain и Ask Wiki работают scale-correctly без cross-domain reference bleed.
Part 4 — SettingВ src/types.ts поле bfsMinScoreRatio: number (default 0.6, диапазон 0..1, 0 = floor off); запись в eval_meta.retrievalConfig. Примечание: 0.6 — доля denseRef, не связанная с наблюдённым denseMax ≈ 0.6; эффективный барьер ≈ 0.6 · denseMax ≈ 0.36. В src/settings.ts слайдер рядом с «BFS fusion». bfsMinScoreRatio — единственный регулятор floor на обоих путях, decoupled от bfsFusion (тот управляет только ordering); ratio = 0 отключает floor везде.
