1. Резюме спецификации

Требования по секциям

#СекцияТребование
R1Part 1 — RetrievalГейт seed-качества сравнивает порог с dense-cosine (denseMax), а не с RRF-fused score; при denseMax ≥ порога используются hybrid-RRF seeds (вектор работает).
R2Part 1 — Observabilitygraph_stats расширен (retrievalMode, denseMax, причина fallback low-similarity/embed-failed); compact-прогресс печатает retrieval-тег; различает «embedding упал» и «score низкий».
R3Part 2 — SettingsНа mobile скрыть ingest/lint-only: «Graph health» (dedup/near-dup/merge-warn) и Chunking. timeouts не трогать.
R4Part 3 — TriggerКнопка Format в mobile-панели view.ts, активна при activeFile && !path.startsWith("!Wiki/").
R5Part 3 — VisionVision на native-agent через vision.enabled + analyzeSingleAttachment (картинки mobile-safe); PDF/Excalidraw — graceful skip по флагу imageOnly.
R6Part 4 — Source-suggestFolderInputSuggest.getSuggestions исключает WIKI_ROOT и его поддерево; закрывает петлю ингеста собственного выхода.

Success criteria (DoD)

ЧастьКритерий приёмки
1Eval-харнес: denseMax ≥ порог → RRF-seeds; < порог → jaccard(low-similarity); throw → embed-failed. Replay лог-запроса → retrievalMode: hybrid, без ложного fallback.
2Ревью гейтов !Platform.isMobile; desktop-раскладка не изменена; ни одна нужная настройка не завёрнута.
3Format картинки на mobile → событие Vision + описание под embed; PDF → одна пометка «Vision skipped (unsupported on mobile)», без падения.
4Юнит getSuggestions исключает любые !Wiki-папки, обычные папки по запросу остаются.
