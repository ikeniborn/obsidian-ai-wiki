# Intent: lint schema context

**Date:** 2026-05-24
**Status:** draft

## Objective
`lint.ts` не читает `.config/_wiki_schema.md` и не передаёт `schema_block` в промпт — LLM проверяет вики без знания конвенций. Нужно исправить. Параллельно — проверить, задействована ли схема в `query.ts`: схема нужна только при изменении вики (ingest, lint, init), но не при запросах. Выявлено в ходе анализа пайплайна промптов агента.

## Desired Outcomes
- `lint.ts` читает `.config/_wiki_schema.md` и передаёт `schema_block` в промпт перед вызовом LLM.
- `query.ts` проверен: если схема там не нужна — она убрана или не добавлена.
- `docs/prompt-architecture.md` обновлена: отражает, какие фазы используют `schema_block`, а какие нет.
- После lint в плагине схема вики сохраняется или актуализируется корректно.

## Health Metrics
- Качество формирования вики не ухудшается (lint-результаты не регрессируют).
- Промпты не раздуваются избыточным контекстом (schema не попадает туда, где не нужна).

## Strategic Context
- Затрагивает: `src/phases/lint.ts`, `src/phases/query.ts`, возможно `src/phases/lint-chat.ts`
- Приоритет: **trust** (качество результата важнее скорости и стоимости)

## Constraints

### Steering (поведенческие)
- Код не должен исключать использование важных инструкций из схемы.
- Реализация должна чётко следовать конвенции передачи `schema_block`.
- Изменения минимальны — только то, что решает проблему.

### Hard (архитектурные)
- Архитектуру (пайплайн, схемы валидации, протокол stream-json) не менять без согласования.
- Избыточные изменения запрещены.

## Autonomy Zones
- **Full autonomy**: чтение файла схемы, передача `schema_block` в промпт lint, обновление docs.
- **Guarded**: изменения в `query.ts` — логировать решение (добавить / убрать / оставить схему).
- **Proposal-first**: любые изменения пайплайна фаз, типов, структуры AgentRunner.
- **No autonomy**: изменения архитектуры stream-json протокола, схем валидации типов.

## Stop Rules
- **Halt if**: изменение затрагивает более трёх файлов или требует изменения типов в `types.ts`.
- **Escalate if**: поведение `query.ts` со схемой неоднозначно — требует решения по дизайну.
- **Done when**: lint передаёт `schema_block`; query проверена и задокументирована; `prompt-architecture.md` обновлена; тесты проходят.
